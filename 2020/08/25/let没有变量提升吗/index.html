let没有变量提升吗

10000 · 2020-8-25 · 次阅读

前言

前段时间，听朋友说let没有变量提升，对此我表示疑问，如果不存在变量提升那let的暂时性死区怎么解释？

变量提升

var 变量提升

MDN上是这么解释的：

从概念的字面意义上说，“变量提升”意味着变量和函数的声明会在物理层面移动到代码的最前面，但这么说并不准确。实际上变量和函数声明在代码里的位置是不会动的，而是在编译阶段被放入内存中。

JavaScript 在执行任何代码段之前，将函数声明放入内存中的优点之一是，你可以在声明一个函数之前使用该函数

函数和变量相比，会被优先提升。这意味着函数会被提升到更靠前的位置。

只有声明被提升

JavaScript 仅提升声明，而不提升初始化。如果你先使用的变量，再声明并初始化它，变量的值将是 undefined

有趣的是，MDN上变量提升的例子里全是var 声明的；

那么总结一下var的变量提升：

var 变量提升仅提升声明且有初始化值 undefined

let 变量提升

我们知道，在使用let声明变量之前，使用变量会报错

我们先看var的例子：

var a = 1;
function test() {
  console.log(a);
};
test()
// 输出：1

我们知道，在函数中使用函数内未定义的变量，会在上层作用域查找该变量，所以这里函数内可以访问函数外部的a

那么我们再函数内加一条a的定义吧：

var a = 1;
function test() {
  console.log(a);
  var a = 2;
};
test()
// 输出：undefined

输出结果符合上述 var 的变量提升

那么我们把函数内部的var声明改为let声明：

var a = 1;
function test() {
  console.log(a);
  let a = 2;
};
test()
// Uncaught ReferenceError: Cannot access 'a' before initialization

唉？报错了，你可能觉得这不是很正常么？但是这却说明了问题

首先报错的意思是：初始化前无法访问 a

假设let没有变量提升，按照变量查找原理，会找到函数外层的变量a并输出

假设let有var变量提升的行为，按照var变量提升的行为来说：

  • 声明提升， 这里满足了，因为这里检测到了函数内部的a定义
  • 声明后赋值 undefined，不满足，因为在这里错误提示变量未初始化

但是上面两种假设都不成立，也就是说，let有变量提升，不过和var不同

这里MDN中let的暂时性死区中解释到：

暂存死区

与通过 var 声明的有初始化值 undefined 的变量不同，通过 let 声明的变量直到它们的定义被执行时才初始化。在变量初始化前访问该变量会导致 ReferenceError。该变量处在一个自块顶部到初始化处理的“暂存死区”中

暂存死区与 typeof

与通过var声明的变量, 有初始化值 undefined和只是未声明的变量不同的是，如果使用typeof检测在暂存死区中的变量, 会抛出ReferenceError异常

总结

var变量提升：

  • 声明提升
  • 声明后赋值 undefined

let变量提升

  • 声明提升
  • 声明后无赋值操作
