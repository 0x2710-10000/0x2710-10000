sass监视文件自动编译

10000 · 2020-9-23 · 次阅读

前言

今天在学sass的时候朋友告诉我可以用 node-sass 使用 node-sass sass文件路径 输出css文件路径 -w 命令监视文件的变化自动编译文件，但是一直在全局环境下没成功安装上 node-sass 于是就想能不能用npm安装的sass直接监视文件变化，于是就找到了如下命令

前提

你已经通过npm 全局(或在项目中)安装好了 sass

sass编译命令

sass编译一个文件：

命令：sass 源scss文件路径:输出的css文件路径

如：sass main.scss:main.css

sass对一个文件进行监控编译

命令：sass 源scss文件路径:输出的css文件路径 --watch

如： sass main.scss:main.css --watch

sass将会监视mian.scss文件的变化并实时编译输出到指定位置的指定文件中

sass对文件夹进行监控编译

命令： sass sass文件夹:css文件夹 --watch

如：sass ./sass:./css --watch

sass将会监视当前目录下scss文件夹下的需要编译成css文件的scss文件并按照目录结构在css目录中生成相应的css文件

提示

亲测编译scss文件时含有中文目录不会报错哦(npm sass 版本： sass@1.26.11)

可以在 package.json 中的 scripts 加入类似以下配置

"scripts": {
  "sass": "sass ./scss:./css --watch"
}

随后可以在项目中运行npm run sass 以启动 sass 对文件或目录的监控
